Overview

nf.io provides a file system abstraction for Virtual Network Function (VNF) management and Orchesration. It exposes the Linux file system interface as the northbound API, enabling user to manage and orchestrate VNF service chains by performing simple file and directory manipulation operations. In this demonstration a sample VNF chain is deployed. The terminal below can be used to collect information about individual VNFs as well as for the entire chain.


Setup

The VNF chain Firewall -- IDS -- Proxy is deployed. The proxy is serving this same page, and can be accessed at this link. Some sample commands to monitor the chain are given below:

Total number of packet drops along a chain: 
find -L /nfio/chns/chain-alpha -name pkt_drops ! -path "*/next/*" ! -path "*/start/*" | xargs cat | awk '{total += $1} END {print total}'
Total transmitted bytes along a chain: 
find -L /nfio/chns/chain-alpha -name tx_bytes ! -path "*/next/*" ! -path "*/start/*" | xargs cat | awk '{total += $1} END {print total}'
List of all VNF instances deployed on a physical machine with IP address 10.0.0.17: 
grep -R '172.17.42.1' /nfio/nf-types
List of all chains a VNF instance is part of: 
find /nfio/chns -lname /nfio/nf-types/firewall/fw-alpha
Total number of active connections to the web proxy (from the logs of a Bro IDS): 
tail -n +8 /nfio/nf-types/ids/ids-alpha/rfs/spool/bro/conn.log | awk '{print $8}' | grep -c 'http'
Access log of web proxy: 
tail -f /nfio/nf-types/proxy/proxy-alpha/rfs/access.log
